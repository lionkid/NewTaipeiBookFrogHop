# 台灣旅行樹蛙 ── 新北市圖書館互動網站
## Claude Code 實作需求說明文件

> **版本**：1.0　**適用對象**：小學 1～6 年級　**部署平台**：Google Sites 內嵌 iframe

---

## 一、專案背景與目標

### 1.1 靈感來源
本專案靈感來自日本手機遊戲「旅かえる（旅行青蛙）」，結合新北市政府出版的三本城市繪本：

| 書號 | 書名 | 主題 |
|------|------|------|
| 書①（紅色）| 《轉角新北練習曲》 | 新北市藝術文化、老街 |
| 書⑧（黃色）| 《金夕窩在新北》 | 新北市煤礦文化（猴硐、九份、平溪） |
| 書⑨（藍色）| 《啟程雲遊新北》 | 新北市自然景觀與節慶 |

### 1.2 核心目標
- 故事架構：樹蛙在書架上的繪本看到好特別的景點，準備要計劃去那裡旅行
- 每天更換一個「今日景點」，讓圖書館裡的翠綠樹蛙前往該地旅行
- 孩子幫樹蛙「打包行李」（選擇適合的旅行物品）
- 透過互動引發孩子對景點的好奇心：例如樹蛙會寄一張有景點照片、自己也在裡面某處的明信片，可在照片裡藏一些樹蛙的腳印讓孩子搜尋
- 最後希望可以鼓勵孩子去書架找出有該景點的繪本閱讀

### 1.3 使用情境
- 地點：學校圖書館的公用電腦（不需登入帳號）
- 使用者：多位孩子輪流使用同一台電腦
- 裝置：桌機螢幕（建議 1024×768 以上），無觸控需求
- 存取方式：老師將網頁嵌入 Google Sites，孩子開啟瀏覽器即可使用

## 二、整體架構

### 2.1 技術規格（重要限制）

```
【部署限制】
- 必須是單一 HTML 檔案（index.html），所有 CSS 與 JS 內嵌其中
- 不可依賴任何需要付費或需要帳號的後端服務
- 不可使用 localStorage（多人共用電腦，不保留個人資料）
- 可使用免費公開 CDN（如 cdnjs、unpkg）

【Google Sites 嵌入方式】
老師將 HTML 檔案上傳至 GitHub Pages 或其他免費靜態主機，
再透過 Google Sites 的「嵌入」功能以 iframe 方式呈現。
建議 iframe 尺寸：寬 900px × 高 650px

【日期邏輯】
景點輪替以「今天的日期」為基準（用 JavaScript Date 物件）。
所有在同一天開啟網頁的孩子，看到的是「同一隻蛙、同一個景點」。
每天自動更換，不需要管理員操作。
```

### 2.2 頁面結構（單頁，無路由切換）

```
┌─────────────────────────────────────────────┐
│  頂部：標題「台灣旅行樹蛙去新北～」+ 今日日期   │
├──────────────────┬──────────────────────────┤
│  左側：樹蛙動畫區  │  右側：景點資訊卡片         │
│  （蛙的狀態圖）   │  - 景點名稱（大字）(今天天氣)    │
│                  │  - 所在書本（書名 + 書號圖示） │
│                  │  - 景點簡介（2～3句，淺白語言）│
│                  │  - 「在書中找找看 🔍」提示    │
├──────────────────┴──────────────────────────┤
│  中間：行李打包區「幫蛙蛙準備行李！」            │
│  （物品格子區，孩子點選物品）                    │
├─────────────────────────────────────────────┤
│  底部：「今天大家幫蛙蛙帶了什麼？」累計統計區      │
└─────────────────────────────────────────────┘
```

```javascript
// 推薦演算法：weatherTags 與 ITEMS 的 tags 有交集者為「推薦」
function getRecommendedItems(location) {
  const allTags = [...location.weather, location.activity];
  return ITEMS.filter(item => item.tags.some(tag => allTags.includes(tag)));
}
```

## 五、每日景點輪替邏輯

```javascript
// 以今天距離某個基準日期的天數，對景點陣列取餘數
function getTodayLocation() {
  const BASE_DATE = new Date('2025-01-01');
  const today = new Date();
  today.setHours(0, 0, 0, 0);
  const dayIndex = Math.floor((today - BASE_DATE) / (1000 * 60 * 60 * 24));
  return LOCATIONS[dayIndex % LOCATIONS.length];
}
```

> **說明**：這確保同一天所有孩子看到相同景點，且每天自動換新，不需要管理員操作。

## 十一、給 Claude Code 的補充說明

1. **孩子年齡考量**：說明文字要口語化、有趣，避免文言或生硬詞彙。蛙蛙說的話要像在跟朋友說話一樣。

2. **教育目的**：每個景點都要清楚告訴孩子「去哪本書找」，這是這個網站最重要的教育功能。

3. **蛙蛙角色**：靈感來自書中的翠綠小蛙角色（戴礦工帽），請用 CSS/SVG 設計一個可愛的綠色蛙蛙，有大圓眼睛、笑臉，可以戴帽子（帽子可根據景點切換，如礦工帽、草帽、毛帽）。

4. **書本顏色系統**：
   - 書① 紅色系（#E84545）：藝術文化
   - 書⑧ 金黃色（#E8B84B）：礦業文化  
   - 書⑨ 藍色系（#4A90D9）：自然景觀

5. **Google Sites 嵌入**：最終產出的 index.html 要能在 `<iframe>` 中正常運作，不得有 `X-Frame-Options: DENY` 等阻止嵌入的 header（靜態 HTML 本身不會有此問題）。

6. **行動裝置不是重點**：雖然要能在手機上基本顯示，但主要優化目標是桌機 1024px 以上寬度，不需要複雜的響應式設計。
